Activer la base de données Firebase

Guide pas-à-pas pour passer du stockage local localStorage à une vraie base de données cloud gratuite Firebase Firestore

Pourquoi Firebase ?

Avant (localStorage)
  • Données stockées dans UN seul navigateur
  • Perdues si le navigateur est effacé
  • Non partagées entre ordinateurs
  • Inaccessibles après formatage
Après (Firebase Firestore)
  • Données stockées dans le cloud Google
  • Survivent aux reformatages et changements de PC
  • Partagées entre tous les administrateurs
  • Gratuit jusqu'à 1 Go / 50 000 lectures par jour
1

Créer un compte et un projet Firebase

  1. Allez sur https://console.firebase.google.com (connexion avec un compte Google)
  2. Cliquez « Créer un projet »
  3. Nom du projet : amicale-attt (ou tout autre nom)
  4. Désactiver Google Analytics (inutile pour ce projet) → Créer le projet
  5. Attendez 30 secondes que le projet soit prêt, puis cliquez Continuer
2

Créer la base de données Firestore

  1. Dans le menu gauche → Build → Firestore Database
  2. Cliquez « Create database »
  3. Choisissez un serveur : sélectionnez europe-west1 (Belgium) (le plus proche de l'Algérie)
  4. Démarrez en mode « Production mode » → Create
  5. Attendez que la base soit initialisée (10-20 secondes)
3

Configurer les règles d'accès Firestore

  1. Dans Firestore → onglet « Rules »
  2. Remplacez tout le contenu par les règles ci-dessous
  3. Cliquez « Publish »
// Règles Firestore — Amicale ATTT
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /site_data/{document} {
      allow read, write: if true;
    }
  }
}
Ces règles permettent l'accès en lecture/écriture depuis le site web. Pour un site interne à l'ATTT, ce niveau de sécurité est suffisant.
4

Obtenir la configuration Firebase

  1. Cliquez sur l'icône ⚙ (Paramètres du projet) en haut à gauche
  2. Onglet « Général » → descendez jusqu'à « Vos applications »
  3. Cliquez « Ajouter une application » → choisissez </> Web
  4. Nom de l'application : Amicale ATTT Site
  5. Ne cochez pas « Configurer Firebase Hosting » → cliquez Enregistrer l'application
  6. Copiez les valeurs de firebaseConfig affichées
// Exemple de configuration affichée par Firebase :
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "amicale-attt.firebaseapp.com",
  projectId: "amicale-attt",
  storageBucket: "amicale-attt.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abc123def456"
};
5

Modifier le fichier js/firebase-config.js

Ouvrez le fichier js/firebase-config.js et remplacez les valeurs par vos propres identifiants Firebase :

/* ─── Activer/Désactiver Firebase ─────────────────────────── */
const FIREBASE_ENABLED = true; /* ← Mettre true */

/* ─── Votre configuration Firebase ─────────────────────────── */
const FIREBASE_CONFIG = {
  apiKey: 'AIzaSy...', // <-- coller vos valeurs
  authDomain: 'amicale-attt.firebaseapp.com',
  projectId: 'amicale-attt',
  storageBucket: 'amicale-attt.appspot.com',
  messagingSenderId: '123456789012',
  appId: '1:123456789012:web:abc123def456'
};
C'est tout ! Sauvegardez le fichier, déployez le site — Firebase est actif. Toutes les saisies (membres, événements, offres, articles) seront automatiquement sauvegardées dans le cloud.
6

Migrer les données existantes (si nécessaire)

Si vous avez déjà des données dans le navigateur (membres, événements créés avant l'activation de Firebase), voici comment les migrer vers le cloud en une seule opération :

  1. Connectez-vous en tant que Maître sur le PC qui contient les données
  2. Ouvrez master.html → section Sauvegarde
  3. Cliquez « Télécharger la sauvegarde (.json) »
  4. Activez Firebase (étapes 1 à 5 ci-dessus) et redéployez le site
  5. Reconnectez-vous en Maître → Sauvegarde → Importer → sélectionnez votre fichier .json → Remplacer toutes les données
Après import, les données sont dans Firestore. Tous les appareils connectés au même projet Firebase les verront automatiquement au prochain chargement de page.
Checklist de vérification
  • Projet Firebase créé sur console.firebase.google.com
  • Firestore Database activé (europe-west1)
  • Règles Firestore publiées (allow read, write: if true)
  • Application Web ajoutée dans les paramètres du projet
  • FIREBASE_ENABLED = true dans js/firebase-config.js
  • Valeurs FIREBASE_CONFIG renseignées correctement
  • Site redéployé
  • Console navigateur (F12) affiche [DB] ✓ Firebase connecté
  • Un membre inscrit depuis l'appareil A est visible depuis l'appareil B
