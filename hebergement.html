Guide d'hébergement du site

Tout ce qu'il faut savoir pour mettre le site en ligne, choisir un hébergeur, configurer un nom de domaine et assurer la maintenance.

1. Introduction

Ce site est 100 % statique

Le site de l'Amicale ATTT est composé uniquement de fichiers HTML, CSS et JavaScript. Il ne nécessite aucun serveur applicatif, aucune base de données, aucun PHP/Python/Node.js.

N'importe quel hébergeur capable de servir des fichiers statiques peut l'héberger — y compris les solutions entièrement gratuites comme GitHub Pages ou Netlify.

2. Prérequis avant déploiement

Avant de mettre le site en ligne, assurez-vous d'avoir complété ces étapes :

Liste de vérification — contenu à personnaliser
Textes : Remplacez tous les textes de démonstration (description de l'association, membres du bureau, etc.) par le vrai contenu.
Images : Ajoutez les vraies photos dans le dossier img/. Remplacez les images placeholder.
Contact : Dans contact.html, mettez la vraie adresse e-mail et activez Formspree ou un autre service de formulaire.
Applications : Remplacez les fichiers placeholder dans assets/app-desktop/ et assets/app-android/ par les vraies applications.
Icônes PWA : Créez img/icon-192.png et img/icon-512.png à partir du logo de l'association (utilisez realfavicongenerator.net).
Comptes admin : Créez les vrais comptes (Administration, Direction, Délégués) et supprimez les comptes de démonstration.
Test local : Testez le site en local sur http://localhost:8080 avant de le déployer (voir commande ci-dessous).

Lancer le serveur local de test

cd "c:\site amicale attt"
python -m http.server 8080
# Puis ouvrez http://localhost:8080 dans votre navigateur

3. Hébergement — GitHub Pages Gratuit

GitHub Pages héberge votre site directement depuis un dépôt Git.
URL fournie : https://votre-org.github.io/amicale-attt/
1
Créez un compte GitHub

Inscrivez-vous sur github.com avec l'adresse e-mail de l'association.

2
Créez un nouveau dépôt (repository)

Cliquez sur New repository. Nommez-le amicale-attt. Cochez Public. Cliquez Create repository.

3
Envoyez vos fichiers

Option A — Interface web (sans Git) :

  • Clic sur uploading an existing file
  • Glissez-déposez tous les fichiers du site
  • Cliquez Commit changes

Option B — Via Git (recommandé pour les mises à jour) :

git init
git add .
git commit -m "Mise en ligne initiale"
git remote add origin https://github.com/votre-org/amicale-attt.git
git push -u origin main
4
Activez GitHub Pages

Dans votre dépôt → Settings → Pages (menu de gauche) → Source = Deploy from a branch → Branch = main → dossier / (root) → Cliquez Save.

5
Votre site est en ligne !

Après 1 à 2 minutes, votre site est accessible à :
https://votre-org.github.io/amicale-attt/

Domaine personnalisé sur GitHub Pages : Dans Settings → Pages → Custom domain, entrez votre domaine (ex. amicale-attt.dz) et créez un enregistrement CNAME chez votre registraire DNS pointant vers votre-org.github.io.

4. Hébergement — Netlify Le plus simple

Netlify permet de déployer le site par simple glisser-déposer.
URL fournie : https://amicale-attt.netlify.app
1
Créez un compte Netlify

Inscrivez-vous sur netlify.com (gratuit, via GitHub, Google ou e-mail).

2
Préparez une archive ZIP

Compressez le dossier entier du site en un fichier .zip (clic droit → Envoyer vers → Dossier compressé).

3
Déployez en glisser-déposer

Sur le tableau de bord Netlify, faites glisser votre .zip dans la zone « Drag and drop your site output folder here ».

4
Votre site est en ligne instantanément !

URL automatique type https://random-name-12345.netlify.app. Vous pouvez la personnaliser dans Site settings → Change site name.

Domaine personnalisé sur Netlify : Domain settings → Add custom domain. Netlify active automatiquement le HTTPS via Let's Encrypt.

5. Hébergement cPanel / FTP Hébergeur classique

Hébergement mutualisé (OVH, Hostinger, Hostens, etc.)
Nécessite l'achat d'un hébergement + nom de domaine (~15–50€/an)
1
Achetez un hébergement et un domaine

Hébergeurs recommandés pour l'Algérie :

  • OVH France — Plans à partir de 1,99€/mois, domaine .com/.fr
  • Hostinger — Économique, panneau Web Hostinger
  • CERIST / ENIC — Pour un domaine .dz (nécessite justificatifs)
2
Connectez-vous à cPanel ou au gestionnaire de fichiers

Accédez à votre hébergement via https://votre-domaine.com/cpanel ou l'URL fournie par votre hébergeur.

3
Via le Gestionnaire de fichiers cPanel
  • Naviguez vers le dossier public_html/
  • Supprimez les fichiers existants (index.php par défaut)
  • Cliquez Téléverser et envoyez le fichier .zip
  • Extrayez l'archive dans public_html/
3b
Via FTP (FileZilla) — méthode alternative 
Hôte    : ftp.votre-domaine.com
Identifiant : (fourni par hébergeur)
Mot de passe: (fourni par hébergeur)
Port    : 21

→ Glissez tous les fichiers du site vers /public_html/
4
Votre site est en ligne

Accessible à https://www.votre-domaine.com (HTTPS après activation SSL).

6. Nom de domaine & DNS

Domaines recommandés

  • .dz — Domaine algérien officiel (via CERIST/ENIC, justificatifs requis, ~2000–5000 TND/an)
  • .com — International, ~10–15€/an (OVH, Namecheap)
  • .org — Pour les associations, ~10–15€/an
  • .netlify.app — Gratuit si hébergé sur Netlify
  • .github.io — Gratuit si hébergé sur GitHub Pages

Enregistrements DNS essentiels

# Pour GitHub Pages
CNAME  www    →  votre-org.github.io
A      @      →  185.199.108.153
A      @      →  185.199.109.153

# Pour Netlify
CNAME  www    →  votre-site.netlify.app

# Pour hébergeur classique
A      @      →  (IP fournie par hébergeur)
CNAME  www    →  @ ou votre-domaine.com

7. HTTPS & Certificat SSL

Le HTTPS est obligatoire pour que la PWA (Application Web Progressive) fonctionne et pour la sécurité de vos membres. Bonne nouvelle : il est gratuit dans la plupart des cas.

GitHub Pages
HTTPS activé automatiquement. Option dans Settings → Pages → Enforce HTTPS.
Netlify
HTTPS activé automatiquement via Let's Encrypt. Aucune configuration requise.
cPanel
Activez le certificat Let's Encrypt depuis cPanel → SSL/TLS → Let's Encrypt SSL.

9. Maintenance & Mises à jour

Mettre à jour le contenu

Pour modifier des pages (texte, photos, événements via l'espace admin) :

  • GitHub Pages : Modifiez les fichiers HTML localement, puis git add . && git commit -m "Mise à jour" && git push. Le site se met à jour automatiquement.
  • Netlify : Re-déposez un nouveau .zip via le tableau de bord, ou liez votre dépôt Git pour des déploiements automatiques.
  • cPanel / FTP : Re-uploadez les fichiers modifiés via FileZilla ou le gestionnaire de fichiers.

Sauvegarde régulière

Sauvegardez régulièrement les données du site :

  • Données membres / contenu : Les données sont dans localStorage du navigateur. Exportez depuis le panneau d'administration (journal, membres).
  • Code source : Conservez une copie du dossier complet sur un disque externe ou dans un dépôt Git privé.
  • Fréquence : Sauvegarde mensuelle recommandée, avant chaque mise à jour majeure.
Firebase intégré ! Les données sont maintenant synchronisées via Firebase Firestore (base de données cloud gratuite de Google). Activez-la en renseignant vos identifiants Firebase dans js/firebase-config.js et en mettant FIREBASE_ENABLED = true. Guide de configuration →

Tâches de maintenance régulières

Chaque mois
  • Vérifier les membres en attente
  • Mettre à jour les événements
  • Publier nouvelles actualités
Chaque trimestre
  • Sauvegarder les données
  • Vérifier tous les liens
  • Renouveler le certificat SSL
Chaque année
  • Renouveler le nom de domaine
  • Renouveler l'hébergement
  • Archiver les anciennes données

  • Le Service Worker (déjà intégré) met en cache les pages pour un chargement quasi-instantané après la première visite
