3 — Tester le site en local

Important : ouvrir les fichiers HTML directement (file://) ne fonctionne pas — le navigateur bloque les modules JS. Il faut un mini-serveur HTTP.

Méthode 1 — Python (recommandé, pré-installé sur Windows 11)

1
Ouvrir un terminal dans le dossier du site (clic droit → Ouvrir dans le terminal)
2
Lancer le serveur : 
python -m http.server 8080
3
Ouvrir dans le navigateur : http://localhost:8080
4
Pour arrêter : appuyer sur Ctrl+C dans le terminal

Méthode 2 — Extension VS Code (Live Server)

1
Dans VS Code → Extensions → chercher Live Server (Ritwick Dey) → Installer
2
Cliquer sur Go Live en bas à droite — le site s'ouvre automatiquement
3
Avantage : rechargement automatique à chaque sauvegarde de fichier

Méthode 3 — Node.js (si installé)

npx serve .
# ou
npx http-server . -p 8080

Ces commandes installent temporairement un serveur HTTP sans rien installer de permanent.

5 — Base de données Cloud (Firebase)

Pourquoi Firebase ?

Par défaut, toutes les données (membres, événements, messages…) sont stockées dans le localStorage du navigateur. Cela signifie que si vous vous connectez depuis un autre PC ou navigateur, les données ne sont pas synchronisées.

Firebase Firestore permet de synchroniser toutes les données en temps réel entre tous les appareils et tous les administrateurs. Le forfait gratuit Spark couvre largement l'usage de l'Amicale.

Activation pas-à-pas

1
Aller sur console.firebase.google.com → Créer un projet → Nom : amicale-attt → Désactiver Analytics → Créer
2
Build → Firestore Database → Create database → europe-west1 (serveur le plus proche) → Mode Production → Enable
3
Onglet Rules → remplacer par ces règles → Publish : 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /site_data/{document} {
      allow read, write: if true;
    }
  }
}
4
Paramètres (⚙) → Général → "Vos applications" → Ajouter une appli Web </> → Copier la configuration affichée
5
Ouvrir js/firebase-config.js et :
  • Remplacer les valeurs de FIREBASE_CONFIG par celles copiées
  • Changer FIREBASE_ENABLED = false en FIREBASE_ENABLED = true
const FIREBASE_ENABLED = true;  // ← activer

const FIREBASE_CONFIG = {
  apiKey:            'AIzaSy...',
  authDomain:        'amicale-attt.firebaseapp.com',
  projectId:         'amicale-attt',
  storageBucket:     'amicale-attt.appspot.com',
  messagingSenderId: '1234567890',
  appId:             '1:1234567890:web:abcdef'
};
6
Sauvegarder et déployer → au premier chargement, les données localStorage sont automatiquement envoyées vers Firestore

Limites du forfait gratuit Firebase (Spark)

1 Go
Stockage données
50K
Lectures / jour
20K
Écritures / jour

Pour une Amicale de quelques centaines de personnes, ces limites ne seront jamais atteintes.

6 — Publier une mise à jour du site

Méthode simple — Modifier directement sur GitHub

1
Aller sur votre dépôt GitHub → naviguer jusqu'au fichier à modifier
2
Cliquer sur l'icône (Edit this file)
3
Faire la modification → Commit changes → Le site est mis à jour automatiquement en 1-2 minutes

Méthode recommandée — Via VS Code + Git

# Dans le dossier du site
git add .
git commit -m "Description de la modification"
git push origin main
# → Site mis à jour en 1-2 minutes sur GitHub Pages
# → Immédiatement sur Netlify

Mettre à jour les applications (PC / Android)

1
Préparer le nouveau fichier .exe ou .apk et le déposer dans assets/app-android/ ou assets/app-desktop/
2
Aller dans master.html → section "Versions app" → remplir le formulaire → télécharger le nouveau version.json
3
Déposer ce version.json dans assets/ sur le serveur
4
Tous les utilisateurs verront une bannière de notification au prochain chargement du site

Checklist avant toute mise à jour

  • Tester en local avant de déployer
  • Faire une sauvegarde des données (master.html → Sauvegarde → Exporter)
  • Vérifier sur mobile (DevTools ou vrai téléphone)
  • Vérifier que FIREBASE_ENABLED est à true si Firebase est activé
  • Tester la connexion avec le compte 347

8 — Gérer les membres

Valider une inscription

  1. admin.html → section Demandes
  2. Cliquer Valider → le membre devient actif
  3. Optionnel : lui envoyer un message de bienvenue via la messagerie

Promouvoir un membre

  1. master.html → section Utilisateurs
  2. Cliquer sur le membre
  3. Changer le Rôle → Enregistrer

Suspendre / Supprimer

  • Suspendre : bloque la connexion sans supprimer les données
  • Supprimer : uniquement depuis master.html — irréversible
  • Le compte 347 est indestructible (protection logicielle)

Comptes Famille

  • Un membre génère un code FAM-XXXX-XXXX dans son profil
  • Sa famille s'inscrit via inscription.html?mode=famille
  • Rôle 0 = accès limité (pas les offres exclusives des membres)
  • Peut consulter événements, conventions, locations

8b — Gérer la galerie photos

Comment ajouter des photos

1
Se connecter avec un compte Délégué (rôle admin) ou supérieur
2
Aller sur galerie.html — le bouton + Ajouter des photos apparaît en haut à droite
3
Choisir un nom d'album (ou en créer un nouveau), ajouter une légende optionnelle, sélectionner les fichiers images
4
Cliquer Enregistrer — les photos sont converties en base64 et stockées dans attt_galerie

Contraintes et conseils

  • Taille max par photo : 2 Mo (affichée sous la sélection)
  • Formats acceptés : JPG, PNG, WebP
  • Les photos sont stockées en base64 dans le navigateur — plus il y en a, plus le localStorage est lourd
  • Conseil : ne pas dépasser 5 Mo total de photos pour éviter les lenteurs — utiliser des photos compressées (ex: squoosh.app)
  • Pour supprimer : bouton "Suppr." sous chaque photo (admin uniquement) ou "Supprimer l'album" entier
  • Les photos sont sauvegardées dans la sauvegarde JSON globale (section Sauvegardes)

Filtrer et naviguer

  • Les boutons d'albums apparaissent automatiquement selon les albums existants
  • Un clic sur une photo l'ouvre en plein écran (Lightbox) avec navigation gauche/droite
  • La recherche texte filtre par nom d'album ou légende

Réinitialiser toutes les données (ATTENTION — irréversible)

Cette action efface tous les membres, événements et données. Faire une sauvegarde avant ! 
// Dans la console DevTools (F12 → Console)
Object.keys(localStorage)
  .filter(k => k.startsWith('attt_'))
  .forEach(k => localStorage.removeItem(k));
location.reload();

10 — Dépannage

Vous avez ouvert le fichier HTML directement (double-clic). Il faut utiliser un serveur HTTP local.
→ Lancer python -m http.server 8080 puis aller sur http://localhost:8080

Le mot de passe est stocké sous forme de hash dans localStorage. Pour le réinitialiser : 
// Console DevTools → copier-coller
const salt='ATTT_AMICALE_DZ_2026';
const s='NOUVEAU_MDP'+salt;
let h=2166136261;
for(let i=0;i>>0;}
const hash=h.toString(16).padStart(8,'0');
const u=JSON.parse(localStorage.getItem('attt_users'));
const idx=u.findIndex(x=>x.login==='347');
u[idx].passwordHash=hash;
localStorage.setItem('attt_users',JSON.stringify(u));
alert('Mot de passe mis à jour');

  • Vérifier que FIREBASE_ENABLED = true dans js/firebase-config.js
  • Vérifier les valeurs de FIREBASE_CONFIG (apiKey, projectId…)
  • Vérifier les règles Firestore (elles doivent autoriser read+write)
  • Ouvrir F12 → Console → chercher les messages [DB]
  • Vérifier que le projet Firebase n'est pas suspendu sur console.firebase.google.com

Le localStorage est lié au navigateur et peut être effacé lors d'un nettoyage. Deux solutions :
1. Firebase activé : les données sont rechargées automatiquement depuis le cloud
2. Sans Firebase : importer la dernière sauvegarde JSON dans master.html → Sauvegarde
